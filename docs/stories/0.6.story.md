## Story

**eu quero** ver **todas as 48 seleções classificadas** (não 42 + 6 placeholders de "Repescagem"),

## Dev Notes

**Por que MERGE em vez de DELETE+INSERT?**
- Tabela `dbo.teams` é referenciada por `matches.home_team_id` e `matches.away_team_id` via FK. DELETE quebraria referências ou exigiria CASCADE complexo.
- MERGE preserva os IDs INT IDENTITY existentes para os 16 teams já presentes (que têm matches ligados no bacpac), e só ADICIONA os 32 faltantes (16 → 48).
- Seguro mesmo se a migration rodar duas vezes (idempotente).

**Por que `Qualified.tsx` é hardcoded e não usa `/api/teams`?**
- Decisão original do projeto. A página tem campos extras (`participations`, `bestResult`, `titles`) que não existem no schema canônico. Para alinhar 100% precisaria adicionar essas colunas ao schema (out of scope desta story — vira tech debt TD-8 se quiser).
- Para esta story: manter hardcoded mas sincronizado com os 48.

**Sobre a página Groups.tsx:**
- Consome `getTeamsByGroup()` de `data/teams.ts` (filtra `!team.isTBD`)
- Texto "(Repescagem)" só aparece se `team.isTBD === true`. Após esta story, nenhum dos 48 terá `isTBD: true && group != null`, então texto some naturalmente.

**Sobre nomes em PT-BR:**
- "Tchéquia" é o nome oficial em português (a forma "República Tcheca" foi substituída em 2016).
- "Bósnia e Herzegovina" é o nome diplomático completo (existe variação "Bósnia-Herzegovina" — usei "e" para coincidir com o estilo dos outros nomes do projeto).
- "Rep. Dem. do Congo" abreviado para caber no card layout (forma completa "República Democrática do Congo" estoura espaço em alguns componentes).
